4. DATA ATTRIBUTES (SVG SECTIONS)

Each SVG <path> section now has:

<path data-section-id="s_101"
      data-section-name="101"
      data-price="$380"
      data-row="A"
      data-seat="seat-s_101-01"
      class="seat">
</path>

Extraction on click:
  seatId = seat.getAttribute('data-seat')        → "seat-s_101-01"
  section = seat.getAttribute('data-section')    → "101"
  row = seat.getAttribute('data-row')            → "A"
  price = seat.getAttribute('data-price')        → "$380"

8. CHECKOUT FLOW

1. USER CLICKS SEAT
   ├─ Event delegation on #map-stage
   ├─ Extract data from [data-*] attributes
   ├─ Check if sold/unavailable → skip
   ├─ Add to cart array
   ├─ Change seat color to #ffe000
   ├─ Add pulse animation
   └─ Sync checkout bar

2. CHECKOUT BAR UPDATES
   ├─ Cart is not empty → show bar
   ├─ Hide old bar → show bar
   ├─ Update count: "Selected: 3 Seats"
   ├─ Update total: "Total: $1140"
   └─ "Pay Now" button clickable

3. USER CLICKS "PAY NOW"
   ├─ openCheckoutModal() called
   ├─ Map cart to itemized HTML
   ├─ Calculate grand total
   ├─ Generate Cash App deep link:
   │  https://cash.app/$NovadeniaConcerts/1140.00
   └─ Display modal

4. CASH APP PAYMENT
   ├─ User opens Cash App app
   ├─ Sends payment to $NovadeniaConcerts
   ├─ Admin receives payment notification
   ├─ Admin confirms via admin-dashboard.html
   │  (manual verification required per design)
   └─ User clicks "Confirm" (manual for now)

5. ORDER CONFIRMATION
   ├─ confirmPurchase() called
   ├─ Clear cart array
   ├─ Reset all seat colors to original
   ├─ Hide checkout bar
   ├─ Close modal
   └─ Order complete 🎉
